# Tags

Tags are lightweight labels you can attach to items across dokuly for categorization and filtering. They are defined per project and can be applied to parts, assemblies, PCBAs, documents, requirements, ECOs, and purchase orders.

---

## How Tags Work

- Tags belong to a **project** — each project has its own set of tags
- An item can have **multiple tags**
- Tags are displayed as colored labels on table rows and item dashboards
- Use tags to filter tables and find items by category

---

## Common Use Cases

- **Component category** — tag parts as "Resistor", "Capacitor", "Connector" for quick filtering
- **Status markers** — tag items as "Needs review", "Prototype", "Legacy"
- **Subsystem grouping** — tag items by subsystem ("Power", "Comms", "Mechanical")
- **Priority** — tag ECOs or requirements by priority level

---

## Creating Tags

Tags are created within a project context. When editing an item's tags, you can select from existing project tags or create new ones.

---

## Filtering by Tags

On any table view that supports tags (parts, documents, ECOs, etc.), use the tag filter to narrow the displayed items to those matching one or more selected tags.
